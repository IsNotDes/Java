Consignes pour le TP3 de CS 312(rendu sur Chamilo)

* Vos codes doivent être un minimum commentés.
* Un code qui ne compile pas ne sera pas corrigé.
* Vous rendrez une archive tgz contenant vos sources dont vos tests, un README et un compte-rendu (PDF).Le Makefile fourni construit une archive avec la bonne forme:
  * éditez le Makefile pour changer MYNAME
  * make tar vous crée une archive correcte
* un exemple de README est disponible dans le repertoire TP3 de Chamilo. 
* Pour le compte rendu il est demandé de fournir un PDF
  - (groupe 3 uniquement) un diagramme de classes de l'application livrée (utiliser draw.io)
  - Dans le compte-rendu, vous copiez collerez une partie de votre code après chaque question (merci de numéroter comme l'énoncé).
  - vous développerez particulierement 1- une analyse interface versus classe abstraite (5 lignes) et 2- votre conception de méthode verifCarte
  - un exemple source de compte-rendu en markdown est fourni. Il n'est pas obligatoire d'utiliser le format markdown pour réaliser le compte rendu (mais si vous le faites, utilisez pandoc pour générer le pdf final)
* les dates sont précisées pour chaque groupe de TP dans la section "rendu".
* pour le groupe 3 on demande en plus un diagramme de classe du projet entier.
